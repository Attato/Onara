# Contributing

## Making changes
You can make changes to the code, fix bugs, add new features, etc. Make sure that your changes comply with the coding standards of the project and follow our recommendations.

## Creating a request to include changes (Pull Request)
When you have finished working on the changes, create a Pull Request. When creating a PR, include information about what you have changed and why these changes are important. Our developers will review your PR and provide feedback or merge it if it meets our standards.

## Feedback and review
Expect feedback and reviews from our developers. We can request changes, add comments, or ask questions. Be ready for discussion and cooperation.

## Merge changes
After your PR has been successfully reviewed and passed all the checks, the project developers will merge your changes into the main branch of the project.
